`TraceEventSchema` in `api/schemas.py:37` has ~50 fields. The frontend `TraceEvent` interface in `frontend/src/types/index.ts:19` only mirrors 12. The frontend is missing:


The frontend accesses event-type-specific data via `event.data` (generic `Record<string, unknown>`), but any code that tries to read typed fields on `TraceEvent` directly will fail silently. Worth verifying this is intentional.
