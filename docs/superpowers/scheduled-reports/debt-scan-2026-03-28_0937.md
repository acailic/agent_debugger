### Findings by Severity

#### HIGH

**1. Dead file: `agent_debugger_sdk/core/decorators.py` — 491 lines**

`agent_debugger_sdk/core/decorators/` is a Python package (directory with `__init__.py`). When both `decorators.py` and `decorators/` exist, Python **always resolves to the package**. Confirmed: `import agent_debugger_sdk.core.decorators` loads `decorators/__init__.py`. The 491-line `decorators.py` is unreachable by any import path. This was likely missed when the package was extracted (recent commit `1fb6eba` removed dead code elsewhere). The file duplicates `trace_llm`, `trace_tool`, `trace_agent` implementations — a maintenance trap if someone edits the wrong file.

**2. 96 skipped tests in `tests/test_replay_depth_l3.py`**

All 96 skips are intentional placeholders for unimplemented L3 features:

**3. `_send_with_retry()` in `agent_debugger_sdk/transport.py:174` — 117 lines**

Single function handles retry logic, timeout, HTTP response parsing, error classification, and fallback behavior. Cyclomatic complexity is high. Candidate for decomposition into smaller helpers.

**4. `trace_llm()` nested structure in `agent_debugger_sdk/core/decorators/llm.py:22`**

`trace_llm` (110 lines) → `decorator` (76 lines) → `async_wrapper` (72 lines). Three deeply nested closures. The sync/async branching logic is tangled inside `async_wrapper`. This mirrors the same structure in the now-dead `decorators.py` — the extraction to a subpackage didn't simplify it.

**5. Implicit API contracts for baseline/drift endpoints**

`api/trace_routes.py:143–180`: `get_agent_baseline` and `get_agent_drift` return `dict[str, Any]` rather than a named Pydantic schema. The frontend has `AgentBaseline` and `DriftResponse` typed interfaces (`frontend/src/types/index.ts`) that must stay manually in sync with the dict structure — no compile-time check. `AnomalyAlertSchema` and `CheckpointDeltaSchema` defined in `api/schemas.py` have no corresponding frontend types.

**6. `aiosqlite` ResourceWarning in test suite**

One test is not closing a database connection properly. The warning (`aiosqlite/core.py:102`) indicates a connection object is GC'd without being explicitly closed. Likely in an async test that doesn't fully await teardown.
