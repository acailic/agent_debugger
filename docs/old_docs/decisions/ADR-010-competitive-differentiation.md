# ADR-010: Competitive Differentiation

**Status:** Under Review
**Date:** 2026-03-23

## Open Challenge

Six differentiators is too many. Dilutes the message. Should pick 1-2 hero features that are immediately demonstrable and easy to explain.

**Action:** Pick the 1-2 features that make someone say "I need this" in under 10 seconds. Likely candidates:
- **"See WHY your agent did that"** (decision provenance) — answers the #1 pain
- **"Replay from any checkpoint"** (time-travel) — visceral, demo-friendly

Everything else (safety, multi-agent, anomaly detection) is a supporting feature, not a headline.

---

## Original Proposal (Deferred)

Six proposed differentiators:
1. Decision-tree debugging
2. Evidence provenance
3. Safety/refusal observability
4. Checkpoint-based replay
5. Multi-agent debugging
6. Real-time anomaly detection

Related to ADR-001 challenge: positioning must be simple enough for a developer to understand in one sentence.
